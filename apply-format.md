# 適用プロンプト＆ルール

## 処理ルール（重要）

### 判定フロー

```
REVIEW.md の各行をチェック：

1. 行が「- [x] #XXX」で始まるか？
   → YES → ステップ 2 へ
   → NO  → スキップ

2. 行に「❌」マークはあるか？
   → YES → スキップ（除外対象）
   → NO  → ステップ 3 へ

3. ユーザーが指示した承認済みID に含まれるか？
   → YES → 実装を反映
   → NO  → スキップ

4. 反映後、REVIEW.md の該当行を「✅ 適用済み」で更新
```

### ルール表

| 記法 | 評価 | 処理 |
|---|---|---|
| `- [x] #001` | ✅ 承認 | **実装反映** |
| `- [x] #001 ❌` | ❌ 除外 | スキップ |
| `- [ ] #002` | ❌ 未決定 | スキップ |
| `- [x] #001 ✅ 適用済み` | ✅ 完了 | 更新のみ |

---

## 実装反映の手順

### 1. REVIEW.md から対象項目を抽出

例：`- [x] #001 climate.kitchen エンティティを climate_entities.yaml に追加`

### 2. 説明に従い、ファイル・内容を修正

climate_entities.yaml に以下を追加：
```yaml
climate.kitchen:
  name: "キッチン気温"
  device_class: temperature
  unit_of_measurement: "°C"
```

### 3. 修正ファイルをリストアップ

反映後、以下の形式で報告：

```
## 実装反映内容

### #001: climate.kitchen エンティティの追加
**ファイル**: climate_entities.yaml
**変更**: climate.kitchen を追加
```

### 4. REVIEW.md を更新

反映完了後、該当行を以下のように更新：

```markdown
変更前:
- [x] #001 climate.kitchen エンティティを追加

変更後:
- [x] #001 climate.kitchen エンティティを追加 ✅ 適用済み
```

また、STATUS コメントを更新：

```markdown
全項目が適用済みの場合:
<!-- STATUS: completed -->

一部のみ適用済みの場合:
<!-- STATUS: partial -->
```

---

## 禁止事項（重要）

- ❌ `- [ ]` の行を処理する（チェックなし = 未決定）
- ❌ `❌` マークの行を処理する（除外対象）
- ❌ 承認されていない項目を勝手に実装反映
- ❌ REVIEW.md の他のセクション（🟡, ✅, 📝）を無視して編集
- ❌ 明らかに仕様違反の実装を加えること

---

## チャット入力例

```
REVIEW.md の [x] 項目のみを、
このドキュメントのルールに従って実装に反映してください。

承認済みID: #001, #002, #003
```

# 適用ルール（apply.sh の動作仕様）

## 判定フロー

```
REVIEW.md の各行をチェック：

1. 行が「- [x] #XXX」で始まるか？
   → NO  → スキップ（未チェック・他のセクション）

2. 行に「❌」マークがあるか？
   → YES → スキップ（ユーザーが除外指定）

3. 行に「✅ 適用済み」があるか？
   → YES → スキップ（すでに反映済み）

4. 実装に反映する
   → 反映後、REVIEW.md の該当行に「✅ 適用済み」を追記
```

## ルール表

| 記法 | 評価 | 処理 |
|---|---|---|
| `- [x] #001 説明` | ✅ 承認 | **実装反映** |
| `- [ ] #001 説明` | ❌ 未決定 | スキップ |
| `- [x] #001 説明 ❌` | ❌ 除外 | スキップ |
| `- [x] #001 説明 ✅ 適用済み` | ✅ 完了 | スキップ（更新のみ） |

---

## 実装反映の手順

### 1. REVIEW.md から承認済み項目を抽出

```bash
grep -E '^- \[x\] #[0-9]+' REVIEW.md | grep -v '❌' | grep -v '適用済み'
```

### 2. 説明に従い実装ファイルを修正

各項目の説明文（ファイル名:行番号 の形式）を参照して修正する。

### 3. REVIEW.md を更新

反映完了後、該当行を更新：

```markdown
変更前:
- [x] #001 climate.kitchen エンティティを追加

変更後:
- [x] #001 climate.kitchen エンティティを追加 ✅ 適用済み
```

STATUS コメントを更新：

```markdown
全項目適用済み  → <!-- STATUS: completed -->
一部のみ適用済み → <!-- STATUS: partial -->
```

### 4. git push

```bash
cd {PROJECT_DIR}
git add -A
git commit -m "apply({LABEL}): {APPROVED_IDS}"
git push
```

---

## サブプロジェクトの適用

サブプロジェクト（`{sub_id}/REVIEW.md`）を適用する場合：

- `REVIEW_FILE` は `{sub_id}/REVIEW.md`
- `implementation_path` はサブプロジェクト固有のパス
- コミットメッセージは `apply({project}/{sub_name}): {APPROVED_IDS}`

---

## 禁止事項

- ❌ `- [ ]` の行（チェックなし）を処理する
- ❌ `❌` マークの行を処理する
- ❌ `✅ 適用済み` の行を再処理する
- ❌ 承認されていない項目を勝手に実装反映する
- ❌ 仕様書（spec.md）を勝手に書き換える
